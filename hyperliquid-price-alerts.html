Hyperliquid Price Alerts Bot

Project Overview

As I got more involved in the crypto space, I found myself regularly checking the price of various tokens I owned every day I woke up and honestly too frequently throughout the day to check the current price. The main hyperliquid UI doesn't provide support for price alerts and since this was the only extra feature I wanted, I didn't want to switch to a different UI. I wanted this to serve two purposes, give me a notification when a price I care about reaches a certain level and to schedule daily notifications at a certain time (i.e. when I wake up). I worked on a similar feature during my time at Wise, although their platform was already built out and all I was doing was making it more stable and provide real-time ingestion. The simplest way to get this up and running was to build a Telegram bot which gets price data from the Hyperliquid API. I'm hoping this will let me sleep better at night.

This was a fun project involving real-time WebSocket data streaming, persistent storage, and building a cooldown system that prevents spam while ensuring users never miss important price movements. I don't expect this to have a large user base so I ended up using SQLite for storage but still tried to take performance into consideration.

Key Features

  • Real-time Price Monitoring: Continuously monitors cryptocurrency prices using Hyperliquid's WebSocket API for instant updates.
  • Telegram Integration: Sends instant notifications via Telegram when price targets are hit with customizable messages.
  • Persistent Storage: SQLite database to store user alerts, settings, and historical data with reliable persistence.
  • Cooldown System: Prevents spam by implementing intelligent cooldown periods for triggered alerts.
  • Multi-token Support: Monitor multiple cryptocurrencies simultaneously with support for all SPOT tokens on Hyperliquid.
  • Cron Alerts: Schedule daily alerts that fire at specific times (e.g., 8am daily) with custom messages.

System Architecture

The application consists of several key components working together to provide reliable price monitoring:

  • AlertService - Manages alert creation, retrieval, and triggering logic
  • CronService - Handles scheduled alerts and cron job management
  • NotificationService - Handles Telegram bot commands and message sending
  • Database - SQLite-based storage for alerts, cron alerts, and user data
  • WebSocket Client - Real-time price monitoring via Hyperliquid API
  • Cron Worker - Background task that triggers scheduled alerts at specified times

How It Works

The bot operates through a simple yet effective workflow:

1. Create an Alert: Users use the /setalert command to set a target price for any supported cryptocurrency on Hyperliquid.

2. Real-time Monitoring: The bot continuously monitors prices via WebSocket connections to Hyperliquid's API for instant updates.

3. Alert Triggering: When the current price reaches your target (within 0.1% tolerance), you'll receive a notification instantly.

4. Cooldown & Reset: After triggering, alerts enter a 1-minute cooldown to prevent spam, then automatically reset for future triggers.

Telegram Commands

The bot supports several commands for managing alerts:

  • /help - Display available commands
  • /alert - View all your current price alerts
  • /setalert <coin> <price> - Create a new price alert (e.g., /setalert HYPE 100.0)
  • /cronalerts - View all your scheduled cron alerts
  • /setcronalert <message> - Create a daily cron alert at 8am
  • /deletecronalert <id> - Delete a cron alert by ID

Future Plans

The project roadmap includes several exciting enhancements:

  • Smart Alerts: Auto-generate alerts based on user's perpetual positions, including liquidation warnings and stop-loss/take-profit levels.
  • Advanced Scheduling: Support for hourly, weekly, and custom cron schedules beyond daily alerts.
  • Performance Metrics: Detailed analytics and performance tracking for alert accuracy and response times.
  • Message Queue: Implement a message queue system for better scalability and reliability at higher volumes.
  • Alert tied to alarm: Send a notification when my phones alarm goes off or I turn it on.
  • Auto expire alerts: Let users choose to auto expire alerts after a certain number of days or after it fires.

The project is open source and available on GitHub. Feel free to explore the code, contribute, or use it as a reference for your own projects.
